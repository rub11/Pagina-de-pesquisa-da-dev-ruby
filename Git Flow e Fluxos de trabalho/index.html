Git Flow e Fluxos de Trabalho

Organizando projetos com estratégia e controle de versões eficiente

O que é Git Flow?

Git Flow é uma estratégia de branching (ramificação) que organiza o desenvolvimento em diferentes tipos de branches com funções específicas. Ideal para projetos com entregas organizadas e múltiplas versões.

Principais Branches no Git Flow

  • main: Contém o código de produção estável.
  • develop: Integração do que será lançado em breve.
  • feature/*: Desenvolvimento de novas funcionalidades.
  • release/*: Preparação de versões para produção.
  • hotfix/*: Correções rápidas em produção.

Fluxo de Trabalho no Git Flow

  1. 1. Criar uma feature:
    git checkout -b feature/nome-da-feature develop
  2. 2. Finalizar a feature:
    git checkout develop
    git merge feature/nome-da-feature
  3. 3. Criar release:
    git checkout -b release/1.0 develop
  4. 4. Finalizar release:
    git checkout main
    git merge release/1.0
    git tag -a v1.0
  5. 5. Corrigir com hotfix:
    git checkout -b hotfix/1.0.1 main

Variações de Fluxo

  • GitHub Flow: Simples e direto, ideal para entregas contínuas. Branch → Pull Request → Merge na main.
  • GitLab Flow: Mistura entre Git Flow e CI/CD. Pode usar merge requests, ambientes e automações.
  • Trunk Based Development: Desenvolvedores trabalham diretamente em uma única branch (geralmente main) com deploys frequentes.

Qual escolher?

A escolha depende do projeto:

  • Projetos grandes: Git Flow ajuda a manter organização.
  • Startups e MVPs: GitHub Flow para agilidade.
  • DevOps e CI/CD: Trunk ou GitLab Flow com pipelines automatizadas.
