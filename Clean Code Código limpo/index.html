Clean Code: Código Limpo

Escrevendo software legível, elegante e sustentável

O que é Clean Code?

Clean Code é um conjunto de boas práticas para escrever códigos legíveis, simples, claros e de fácil manutenção. A ideia é que qualquer desenvolvedor consiga entender o código como se estivesse lendo um texto bem escrito.

1. Bons nomes para variáveis e funções

Evite abreviações e nomes genéricos. Prefira nomes descritivos que explicam a função da variável ou método.

// Ruim
int a = c * d;

// Melhor
int quantidadeDeHorasTrabalhadas = cargaHoraria * diasTrabalhados;

2. Funções pequenas e coesas

Cada função deve fazer apenas uma coisa. Funções curtas são mais fáceis de testar, entender e manter.

// Ruim
function processarPedido(pedido) {
  validar(pedido);
  calcularDesconto(pedido);
  salvarNoBanco(pedido);
  enviarEmail(pedido);
}

// Melhor
function processarPedido(pedido) {
  validarPedido(pedido);
  aplicarDesconto(pedido);
  persistirPedido(pedido);
  notificarCliente(pedido);
}

3. Evite comentários desnecessários

Comentários são úteis, mas o código limpo deve se explicar sozinho. Prefira melhorar o código do que explicar um código ruim.

// Comentário desnecessário
// Soma os valores do array
let soma = array.reduce((a, b) => a + b, 0);

// Melhor sem o comentário, pois o código é claro
let totalDeItens = array.reduce((a, b) => a + b, 0);

4. Remova código morto

Não mantenha código comentado ou que não é mais utilizado. Isso polui o projeto e causa confusão.

// Código antigo comentado (evitar)
/* function antigoLogin() {
  // ...
} */

5. Trate erros com clareza

Evite deixar mensagens genéricas. Mostre exatamente o que aconteceu e por quê.

// Ruim
throw new Error("Erro");

// Melhor
throw new Error("Usuário não encontrado com o ID fornecido.");

Conclusão

Clean Code não é apenas um conceito, é uma filosofia. Escrever código limpo é uma demonstração de respeito aos colegas desenvolvedores, à equipe e a você mesmo no futuro.
