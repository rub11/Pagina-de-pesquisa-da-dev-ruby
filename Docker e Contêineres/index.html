Docker e Contêineres

Virtualização leve e portabilidade de aplicações

O que é Docker?

Docker é uma plataforma que permite empacotar aplicações e suas dependências em contêineres — unidades leves e portáveis que garantem que o software funcione em qualquer ambiente.

O que são Contêineres?

Contêineres são ambientes isolados que compartilham o mesmo sistema operacional, mas que mantêm bibliotecas e configurações independentes. São mais rápidos e eficientes que máquinas virtuais.

  • Leves e rápidos para iniciar
  • Reproduzíveis e consistentes
  • Fáceis de versionar e compartilhar

Benefícios do Docker

  • Portabilidade: "Funciona na minha máquina" deixa de ser um problema.
  • Escalabilidade: Facilita o deploy de múltiplas instâncias.
  • Produtividade: Reduz o tempo gasto com configurações de ambiente.
  • Integração com CI/CD: Ideal para pipelines automatizadas.

Comandos Docker Básicos

Alguns comandos úteis para começar:

docker build -t minha-imagem . docker run -d -p 3000:3000 minha-imagem docker ps docker stop <id_do_container> docker images docker rmi <nome_da_imagem>

Dockerfile

Um Dockerfile define o ambiente de execução da aplicação. Exemplo:

FROM node:18 WORKDIR /app COPY . . RUN npm install EXPOSE 3000 CMD ["npm", "start"]

Docker Hub

Docker Hub é um repositório público onde você pode publicar e baixar imagens Docker.

  • docker pull nginx baixa a imagem oficial do NGINX
  • docker push usuario/minha-imagem envia uma imagem para sua conta

Docker Compose

Docker Compose permite orquestrar múltiplos contêineres com um único arquivo YAML:

version: '3' services: app: build: . ports: - "3000:3000" db: image: postgres environment: POSTGRES_PASSWORD: exemplo

Quando usar Docker?

Docker é ideal quando você precisa de:

  • Ambientes padronizados
  • Microserviços
  • Deploy rápido
  • Testes isolados
