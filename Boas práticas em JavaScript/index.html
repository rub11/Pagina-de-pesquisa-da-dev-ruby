Boas Práticas em JavaScript

Escrevendo código limpo, seguro e eficiente com estilo

1. Use const e let no lugar de var

var possui escopo de função e pode causar bugs sutis. Use const para variáveis imutáveis e let para variáveis que serão reatribuídas.

const nome = "Ruby";
let idade = 28;

2. Nomeie variáveis de forma clara

Evite nomes genéricos como x, temp ou data. Prefira nomes descritivos:

let precoProduto = 99.90;
let totalCarrinho = precoProduto * 3;

3. Evite código duplicado

Crie funções reutilizáveis sempre que possível:

function formatarPreco(valor) {
  return "R$ " + valor.toFixed(2);
}

4. Prefira funções puras

Funções que não alteram variáveis externas são mais previsíveis e fáceis de testar.

5. Documente com clareza

Comentários claros ajudam a manter o código compreensível para outros devs (e você no futuro):

// Função para calcular o desconto de um produto
function calcularDesconto(valor, percentual) {
  return valor - (valor * (percentual / 100));
}

6. Trate erros corretamente

Utilize try...catch para capturar exceções de forma segura:

try {
  JSON.parse(dados);
} catch (erro) {
  console.error("Erro ao analisar os dados:", erro.message);
}

7. Utilize Arrow Functions

Elas são mais sucintas e respeitam o escopo do this:

const saudacao = nome => `Olá, ${nome}!`;

8. Evite grandes blocos de código

Divida lógicas complexas em funções menores e específicas.

9. Use Desestruturação

Facilita o acesso a propriedades de objetos:

const usuario = { nome: "Ruby", idade: 28 };
const { nome, idade } = usuario;

10. Ferramentas recomendadas

  • ✅ ESLint: análise estática de código
  • ✅ Prettier: formatação automática
  • ✅ JSDoc: documentação de funções
  • ✅ Jest: testes unitários

Conclusão

Boas práticas fazem toda a diferença em projetos colaborativos e escaláveis. Um código limpo é um presente para seu futuro eu — e para o time também.
