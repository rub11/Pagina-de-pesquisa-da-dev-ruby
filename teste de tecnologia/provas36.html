Quiz sobre Design de Sistemas

1. O que é Design de Sistemas?

  • Processo de criação de um sistema para resolver problemas específicos
  • Apenas o design visual de interfaces
  • A construção de servidores para aplicações

2. Quais são as principais etapas do Design de Sistemas?

  • Análise, Design, Implementação e Manutenção
  • Apenas a análise de requisitos
  • Desenvolvimento de interfaces gráficas

3. O que define a arquitetura de um sistema?

  • A estrutura de componentes e suas interações
  • Somente o design da interface de usuário
  • A escolha da linguagem de programação

4. O que são requisitos não funcionais?

  • Requisitos relacionados à performance, segurança e usabilidade
  • Requisitos relacionados ao design visual
  • Requisitos para a configuração do servidor

5. Qual é a diferença entre análise e design de sistemas?

  • Análise define os requisitos, e design cria soluções para atendê-los
  • Análise é sobre escrever código, design é sobre a estética
  • Não há diferença entre análise e design

6. O que é uma arquitetura monolítica?

  • Uma arquitetura onde todos os componentes são interligados em um único bloco
  • Uma arquitetura baseada em microservices
  • Uma arquitetura distribuída com muitos servidores independentes

7. O que é uma API em sistemas?

  • Interface de Programação de Aplicações, que permite a comunicação entre sistemas
  • Uma interface gráfica para o usuário final
  • Um tipo de banco de dados relacional

8. O que são sistemas distribuídos?

  • Sistemas que utilizam múltiplos computadores para processar informações
  • Sistemas que rodam em um único computador
  • Sistemas focados apenas na interação com o usuário

9. Qual é a principal vantagem de sistemas baseados em nuvem?

  • Escalabilidade e flexibilidade
  • Maior controle sobre hardware
  • Acesso mais rápido à memória local

10. O que é um Diagrama de Fluxo de Dados (DFD)?

  • Um modelo gráfico que representa o fluxo de informações em um sistema
  • Um diagrama que descreve a estrutura de um banco de dados
  • Um tipo de diagrama de interfaces de usuário
