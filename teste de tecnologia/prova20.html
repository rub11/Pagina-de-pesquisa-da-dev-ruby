Quiz sobre Arquitetura de Software

1. O que é Arquitetura de Software?

A estrutura organizacional de um sistema de software, incluindo a interação dos componentes
A escolha das linguagens de programação para um projeto
A implementação do código-fonte do software

2. Qual é a principal responsabilidade de um arquiteto de software?

Definir a estrutura e as diretrizes para o desenvolvimento do software
Codificar as funcionalidades principais do sistema
Gerenciar a equipe de desenvolvimento

3. Qual é a diferença entre Arquitetura de Software e Design de Software?

Arquitetura se foca na estrutura geral, enquanto o design detalha a implementação dos componentes
Arquitetura é apenas a escolha de tecnologias, enquanto o design é sobre como o sistema vai ser implementado
Não existe diferença, ambos se referem ao mesmo processo

4. O que é uma arquitetura monolítica?

Uma arquitetura em que todos os componentes do software são integrados em uma única aplicação
Uma arquitetura baseada em microserviços, com componentes independentes
Uma arquitetura de sistema distribuído em várias máquinas físicas

5. O que são padrões de arquitetura?

Soluções comprovadas para problemas recorrentes em projetos de software
Estilos de programação que garantem a melhor performance de software
Estruturas de hardware para suportar aplicações de software

6. Qual é a vantagem de usar a arquitetura baseada em microserviços?

Facilidade de escalabilidade e manutenção de serviços independentes
Aumento da complexidade na implementação de um sistema único
Melhora no desempenho da aplicação em um único servidor

7. O que é um diagrama de arquitetura de software?

Um gráfico que mostra a estrutura e interação dos componentes do sistema
Uma lista detalhada de todas as bibliotecas usadas no projeto
Um mapa de rede de computadores conectados ao sistema

8. Qual é a principal característica da arquitetura em camadas?

A separação do sistema em camadas que tratam de funcionalidades específicas, como apresentação, negócios e dados
A arquitetura não tem camadas específicas, tudo é feito em uma única camada de dados
A separação de responsabilidades com foco em microservices

9. O que é escalabilidade em uma arquitetura de software?

A capacidade do sistema de lidar com um aumento de carga sem perda de performance
A capacidade do sistema de ser acessado por diferentes tipos de dispositivos
A capacidade do sistema de rodar em diferentes linguagens de programação

10. O que é uma arquitetura orientada a eventos?

Uma arquitetura onde os componentes do sistema se comunicam por eventos assíncronos
Uma arquitetura baseada em chamadas síncronas entre componentes
Uma arquitetura que utiliza processos em lote para comunicação entre sistemas
