Quiz sobre Agile e Scrum

1. O que é o Scrum?

Uma metodologia ágil para gerenciamento de projetos
Uma ferramenta de automação de tarefas empresariais
Uma técnica de desenvolvimento de software apenas

2. Quais são os principais papéis no Scrum?

Product Owner, Scrum Master e Team Members
CEO, Diretor de TI, Gerente de Projeto
Líder de Desenvolvimento, Programador, Analista de Sistemas

3. O que é um Sprint?

Um ciclo de desenvolvimento que dura de 1 a 4 semanas
Uma reunião de planejamento semanal
A reunião diária do Scrum

4. Qual é o objetivo de uma reunião diária no Scrum (Daily Standup)?

Acompanhar o progresso e discutir obstáculos
Planejar as próximas entregas de longo prazo
Decidir sobre as prioridades do projeto

5. O que é um Product Backlog?

Uma lista priorizada de funcionalidades e melhorias para o produto
Uma lista de tarefas de curto prazo que a equipe vai realizar
Um documento detalhado com especificações técnicas

6. Quem é o responsável por manter o Product Backlog?

O Product Owner
O Scrum Master
Os desenvolvedores

7. O que é o Incremento no Scrum?

O resultado do Sprint, uma nova versão do produto
O trabalho feito durante o Sprint, mas ainda incompleto
Uma lista de todas as funcionalidades do produto

8. Qual é a principal vantagem do uso de metodologias ágeis como o Scrum?

A flexibilidade para mudanças e entregas rápidas
A centralização das decisões no Scrum Master
A busca por perfeição nas primeiras versões do produto

9. O que é o papel do Scrum Master?

Garantir que o processo Scrum seja seguido corretamente
Realizar as entregas do produto para o cliente
Desenvolver o código do sistema

10. O que é o Scrum na prática?

Um framework ágil para gestão de projetos complexos
Uma ferramenta para criação de planos de projeto detalhados
Um sistema de gestão de tarefas para equipes de TI
