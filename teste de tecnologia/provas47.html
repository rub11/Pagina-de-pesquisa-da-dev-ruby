Quiz sobre Infraestrutura como Código

1. O que é Infraestrutura como Código (IaC)?

  • É a prática de gerenciar e provisionar infraestrutura através de código em vez de processos manuais.
  • É a prática de construir a infraestrutura de TI com base em modelos de redes locais tradicionais.
  • É uma metodologia que utiliza apenas ferramentas de virtualização sem automação.

2. Qual a principal vantagem da Infraestrutura como Código?

  • Permite a automação do provisionamento, gerenciamento e configuração da infraestrutura.
  • Aumenta a complexidade e o custo dos projetos de TI, tornando-os mais difíceis de gerenciar.
  • Reduz a escalabilidade dos sistemas, pois não é possível automatizar processos com IaC.

3. Quais ferramentas são usadas para implementar Infraestrutura como Código?

  • Ferramentas como Terraform, Ansible, Puppet, e Chef são amplamente utilizadas para IaC.
  • Ferramentas como o Kubernetes são exclusivas para IaC, mas não são a melhor escolha.
  • Infraestrutura como Código pode ser feita apenas com scripts manuais de shell.

4. O que significa a expressão "Desenvolvimento DevOps" no contexto de IaC?

  • Refere-se à integração contínua entre desenvolvimento e operações, favorecendo o uso de IaC para automação e colaboração.
  • Refere-se a um modelo de desenvolvimento onde apenas as operações são automatizadas, sem integração com o desenvolvimento.
  • É um conceito que não tem relação com IaC, mas sim com o desenvolvimento de interfaces de usuário.

5. O que significa "Imutabilidade" em IaC?

  • Imutabilidade significa que, uma vez que a infraestrutura é criada, ela não deve ser alterada manualmente.
  • Imutabilidade significa que a infraestrutura é criada de forma programável, mas ainda pode ser modificada manualmente conforme necessário.
  • Imutabilidade significa que toda a infraestrutura precisa ser gerida apenas por scripts de automação.

6. Quais são os benefícios do uso de IaC para o provisionamento de servidores em nuvem?

  • A automação, escalabilidade e consistência são benefícios chave do uso de IaC para servidores em nuvem.
  • IaC ajuda a criar uma rede local tradicional para servidores em nuvem, mas não oferece benefícios significativos para escalabilidade.
  • A única vantagem é que facilita o provisionamento manual de servidores em nuvem.

7. O que são "Templates" no contexto de IaC?

  • Templates são arquivos de código que definem a infraestrutura desejada, podendo ser reutilizados para criar ambientes idênticos.
  • Templates são scripts utilizados apenas para monitoramento de infraestrutura em tempo real.
  • Templates são utilizados para escrever códigos de interfaces de usuário, não relacionados a infraestrutura.

8. Quais são os principais desafios da Infraestrutura como Código?

  • Os principais desafios incluem gerenciamento de versões, complexidade de configuração e necessidade de monitoramento contínuo.
  • Os principais desafios são a falta de ferramentas e a incompatibilidade com nuvens públicas.
  • Não há grandes desafios associados a IaC, ela é uma solução simples de implementar e manter.

9. O que é "Orquestração" no contexto de IaC?

  • Orquestração refere-se à automação e coordenação de múltiplos serviços e recursos de infraestrutura para garantir que trabalhem juntos.
  • Orquestração é o processo manual de definir as configurações de servidores em um único local.
  • Orquestração é o processo de automatizar a implementação de front-end, mas não tem relação com a infraestrutura.

10. O que é "State File" em IaC?

  • State File é um arquivo que mantém o estado atual da infraestrutura, usado para gerenciar alterações e provisionamentos subsequentes.
  • State File é um arquivo que contém as configurações de segurança para acesso a servidores e redes.
  • State File é um arquivo de log que registra a execução do código sem impacto nas configurações.
