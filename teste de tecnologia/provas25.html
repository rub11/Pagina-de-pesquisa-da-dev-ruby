Quiz sobre Microserviços

O que são microserviços?

  • Uma arquitetura de software que divide um aplicativo em serviços independentes e autônomos.
  • Um modelo de banco de dados distribuído para escalabilidade.
  • Uma tecnologia de containers que executa múltiplos serviços em um único ambiente.

Qual é uma das principais vantagens dos microserviços?

  • Escalabilidade independente, permitindo que cada serviço seja dimensionado conforme necessário.
  • Necessidade de menos servidores para funcionar eficientemente.
  • Menor custo operacional devido à centralização do gerenciamento de sistemas.

Qual é a principal desvantagem do uso de microserviços?

  • A complexidade na gestão e integração de vários serviços independentes.
  • O baixo desempenho em comparação com sistemas monolíticos.
  • A falta de escalabilidade devido ao número limitado de microserviços.

Qual tecnologia é frequentemente utilizada para implementar microserviços?

  • Docker, para criação de containers que isolam os microserviços.
  • Redis, para cache de dados entre os microserviços.
  • MongoDB, como banco de dados relacional para comunicação entre os microserviços.

O que é a comunicação síncrona entre microserviços?

  • Quando os microserviços trocam mensagens em tempo real e esperam uma resposta imediata.
  • Quando os microserviços comunicam-se sem necessidade de esperar uma resposta.
  • Quando os microserviços fazem uso de filas de mensagens para comunicação assíncrona.

O que é a comunicação assíncrona entre microserviços?

  • Quando os microserviços trocam mensagens sem esperar por uma resposta imediata, como em filas de mensagens.
  • Quando os microserviços trocam mensagens e sempre aguardam uma resposta antes de continuar.
  • Quando os microserviços comunicam-se por meio de chamadas HTTP diretas.

O que significa "failover" em um sistema de microserviços?

  • Mecanismo de backup que assegura a continuidade do sistema em caso de falha de um microserviço.
  • A habilidade do microserviço de escalar automaticamente quando a carga aumenta.
  • O processo de rodar múltiplas instâncias do microserviço para distribuir a carga de trabalho.

O que é a "orquestração de microserviços"?

  • Coordenar e gerenciar a comunicação entre microserviços para garantir seu funcionamento adequado.
  • A organização do código-fonte dos microserviços em um único repositório.
  • A escolha de um microserviço central para controlar todos os outros serviços de forma independente.

Qual é a principal diferença entre um monolito e uma arquitetura de microserviços?

  • Em um monolito, o sistema é uma única aplicação, enquanto em microserviços o sistema é dividido em componentes independentes.
  • Em um monolito, os componentes são distribuídos entre várias máquinas, enquanto em microserviços os componentes rodam em um único servidor.
  • Em um monolito, todos os serviços são escaláveis independentemente, enquanto em microserviços não há essa possibilidade.

O que é o "CI/CD" em um ambiente de microserviços?

  • Integração contínua e entrega contínua, práticas que automatizam o desenvolvimento, testes e implementação de microserviços.
  • Controle de informações e desenvolvimento colaborativo entre microserviços.
  • Um processo manual de deploy de cada microserviço para a produção.
