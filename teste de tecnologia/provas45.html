Quiz sobre Engenharia de Software Ágil

1. O que é Engenharia de Software Ágil?

  • É um conjunto de práticas de desenvolvimento de software que enfatizam a entrega incremental e iterativa.
  • É um processo rígido e planejado que segue um cronograma bem definido.
  • É uma metodologia que foca em minimizar a comunicação entre as equipes.

2. Qual é o objetivo principal do Scrum dentro da Engenharia de Software Ágil?

  • Facilitar a entrega de software funcional em ciclos curtos chamados sprints.
  • Garantir que o projeto seja completado dentro do prazo estimado, sem alterações.
  • Focar apenas na documentação do software, sem se preocupar com a funcionalidade.

3. Qual é o papel do Product Owner no Scrum?

  • Ser o responsável por priorizar o backlog do produto e garantir que a equipe entenda as necessidades do cliente.
  • Gerenciar a equipe de desenvolvimento e garantir que todos os membros cumpram os prazos.
  • Fazer a documentação técnica de todas as etapas do projeto.

4. O que significa "Sprint" no contexto de Scrum?

  • É um ciclo de trabalho de duração fixa, geralmente de 1 a 4 semanas, para concluir uma parte do projeto.
  • É uma reunião onde toda a equipe define os requisitos do projeto.
  • É um documento formal que descreve todas as funcionalidades do software.

5. O que é o Backlog do Produto?

  • É uma lista priorizada de funcionalidades e melhorias que devem ser feitas no produto.
  • É um plano de marketing para a divulgação do software.
  • É um conjunto de documentos legais que detalham o contrato de desenvolvimento.

6. Qual a principal vantagem do uso de Metodologias Ágeis?

  • Permite uma adaptação rápida às mudanças nos requisitos do cliente.
  • Assegura que todo o projeto seja entregue sem alterações no escopo original.
  • Foca em produzir uma documentação extensa antes de iniciar o desenvolvimento.

7. O que significa a expressão "Iteração" na Engenharia de Software Ágil?

  • É o processo de repetir ciclos de desenvolvimento de software, melhorando a cada nova iteração.
  • É uma técnica para acelerar a entrega de software sem garantir a qualidade.
  • É o momento de revisar todo o código do software de uma vez para garantir que não há erros.

8. O que é a reunião diária (Daily Standup) no Scrum?

  • É uma reunião rápida onde a equipe compartilha o progresso e identifica bloqueios no trabalho.
  • É uma reunião formal para discussão de questões legais do projeto.
  • É uma reunião de planejamento do próximo ciclo de desenvolvimento.

9. O que é a "Revisão de Sprint"?

  • É uma reunião realizada ao final de cada sprint, onde a equipe apresenta o que foi desenvolvido.
  • É uma reunião para discutir o plano de marketing do produto.
  • É uma reunião para revisar os custos do projeto.

10. O que é o papel do Scrum Master?

  • É o responsável por remover impedimentos, garantindo que a equipe siga os processos do Scrum corretamente.
  • É o responsável por definir os requisitos do projeto com o cliente.
  • É o responsável por realizar todas as tarefas de desenvolvimento do projeto.
