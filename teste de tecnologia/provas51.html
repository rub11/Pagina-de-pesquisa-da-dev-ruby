Quiz sobre Comunicação via APIs

1. O que é uma API?

  • API (Interface de Programação de Aplicações) é um conjunto de definições e protocolos que permite que sistemas diferentes se comuniquem entre si.
  • API é uma ferramenta que permite a comunicação entre o hardware do computador e o sistema operacional.
  • API é uma linguagem de programação específica para web, que facilita a criação de sites dinâmicos.

2. O que significa REST em comunicação de APIs?

  • REST (Representational State Transfer) é um estilo arquitetônico que utiliza métodos HTTP para comunicação, como GET, POST, PUT e DELETE.
  • REST significa Real-time Extension Server Technology, e é usado para comunicação em tempo real entre sistemas.
  • REST é um tipo de banco de dados usado para armazenar dados em uma API.

3. O que é uma solicitação HTTP?

  • Uma solicitação HTTP é uma requisição enviada de um cliente para um servidor, pedindo algum recurso ou realizando alguma ação.
  • Uma solicitação HTTP é uma requisição entre dois servidores para estabelecer uma conexão de rede.
  • Uma solicitação HTTP é uma requisição para transferir dados entre dois aplicativos no mesmo sistema operacional.

4. O que é JSON em comunicação via API?

  • JSON (JavaScript Object Notation) é um formato de dados leve e legível por humanos, frequentemente usado para transmitir dados entre cliente e servidor.
  • JSON é uma tecnologia de compressão usada para reduzir o tamanho das requisições API.
  • JSON é uma linguagem de programação usada para desenvolver APIs em sistemas de grande porte.

5. O que é o método GET em uma API?

  • O método GET é usado para solicitar dados de um servidor sem realizar alterações nos dados do servidor.
  • O método GET é utilizado para enviar dados ao servidor, alterando os dados existentes.
  • O método GET é usado apenas para buscar arquivos de mídia em APIs de sites de streaming.

6. O que é o cabeçalho (header) em uma solicitação de API?

  • O cabeçalho contém informações adicionais sobre a requisição, como o tipo de conteúdo e as credenciais de autenticação.
  • O cabeçalho contém dados da resposta que serão exibidos na interface do usuário.
  • O cabeçalho é utilizado para definir as permissões de acesso do servidor à API.

7. O que é uma resposta HTTP?

  • Uma resposta HTTP é a resposta do servidor à solicitação do cliente, que pode incluir dados ou uma mensagem de status.
  • Uma resposta HTTP é a resposta do cliente para um erro de requisição não encontrada no servidor.
  • Uma resposta HTTP é um protocolo que garante a comunicação entre dois servidores de APIs diferentes.

8. O que é OAuth em comunicação via APIs?

  • OAuth é um protocolo de autorização usado para permitir que um aplicativo acesse recursos de outro aplicativo em nome do usuário.
  • OAuth é uma biblioteca de criptografia usada para proteger a comunicação em APIs.
  • OAuth é um framework de segurança utilizado para validar chaves de acesso em APIs de serviços bancários.

9. O que significa a sigla API RESTful?

  • API RESTful refere-se a APIs que seguem os princípios de REST, utilizando métodos HTTP e focando em recursos e operações simples.
  • API RESTful é uma arquitetura que utiliza uma única requisição para obter todos os dados necessários em uma aplicação.
  • API RESTful é uma tecnologia usada apenas para comunicação entre aplicativos móveis e servidores de dados.

10. O que é um token de acesso em uma API?

  • Um token de acesso é um código gerado para permitir que o usuário acesse recursos protegidos de uma API, após autenticação.
  • Um token de acesso é um identificador único de cada servidor que comunica via API.
  • Um token de acesso é uma chave de criptografia usada para ocultar os dados transmitidos nas APIs.
