Kubernetes e Orquestração

Gerencie contêineres em larga escala com eficiência

O que é Kubernetes?

Kubernetes, também conhecido como K8s, é uma plataforma de código aberto desenvolvida pelo Google para orquestrar, escalar e gerenciar contêineres. Ele automatiza tarefas como deploy, balanceamento de carga, monitoramento e recuperação de falhas.

Por que usar Kubernetes?

  • Escalabilidade: Lida com milhares de contêineres automaticamente.
  • Alta disponibilidade: Replica aplicações para garantir resiliência.
  • Deploy contínuo: Facilitado com integrações de CI/CD.
  • Gerenciamento centralizado: Administre tudo com um único cluster.

Componentes do Kubernetes

  • Pod: A menor unidade, contém um ou mais contêineres.
  • Node: Servidor onde os Pods são executados.
  • Cluster: Conjunto de Nodes gerenciados.
  • Service: Abstrai o acesso aos Pods, com IP fixo e load balancing.
  • Deployment: Gerencia o ciclo de vida dos Pods (criação, update, rollback).

Exemplo básico de Deployment

apiVersion: apps/v1 kind: Deployment metadata: name: minha-app spec: replicas: 3 selector: matchLabels: app: minha-app template: metadata: labels: app: minha-app spec: containers: - name: minha-app image: node:18 ports: - containerPort: 3000

Ferramentas complementares

  • Helm: Gerenciador de pacotes para Kubernetes
  • Prometheus + Grafana: Monitoramento e visualização de métricas
  • Ingress Controller: Gerencia acesso HTTP/HTTPS aos serviços

Comandos kubectl úteis

kubectl get pods kubectl get services kubectl describe pod <nome> kubectl apply -f deployment.yaml kubectl delete pod <nome>

Quando usar Kubernetes?

  • Projetos que exigem alta disponibilidade
  • Aplicações distribuídas em microserviços
  • Ambientes de produção e testes escaláveis
  • Necessidade de gerenciamento automatizado
