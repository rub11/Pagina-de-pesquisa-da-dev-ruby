Programação Orientada a Objetos

O que é Programação Orientada a Objetos?

A Programação Orientada a Objetos (POO) é um paradigma de programação que organiza o software em objetos, que são instâncias de classes. Cada objeto encapsula dados e comportamentos relacionados, promovendo modularidade, reutilização e manutenção mais fácil do código.

Conceitos Fundamentais

  • Classe: Um molde que define atributos e métodos comuns a todos os objetos do mesmo tipo.
  • Objeto: Uma instância de uma classe, representando uma entidade com estado e comportamento.
  • Encapsulamento: Restrição do acesso direto a alguns componentes de um objeto, promovendo a integridade dos dados.
  • Herança: Capacidade de uma classe derivar propriedades e comportamentos de outra classe.
  • Polimorfismo: Capacidade de diferentes classes responderem de maneira distinta a uma mesma mensagem ou método.
  • Abstração: Foco nos aspectos essenciais de uma entidade, ocultando detalhes desnecessários.

Exemplo Prático em Java

public class Animal {
    private String nome;

    public Animal(String nome) {
        this.nome = nome;
    }

    public void emitirSom() {
        System.out.println("Som genérico de animal");
    }
}

public class Cachorro extends Animal {
    public Cachorro(String nome) {
        super(nome);
    }

    @Override
    public void emitirSom() {
        System.out.println("Latido");
    }
}

Neste exemplo, a classe Cachorro herda de Animal e sobrescreve o método emitirSom, demonstrando herança e polimorfismo.

Vantagens da POO

  • Reutilização de código: Através da herança e composição.
  • Facilidade de manutenção: Código modular e organizado.
  • Maior segurança: Encapsulamento protege os dados internos.
  • Facilidade de modelagem: Representação mais próxima do mundo real.

Desvantagens da POO

  • Complexidade inicial: Curva de aprendizado mais acentuada.
  • Overhead: Pode haver consumo adicional de recursos.
  • Design inadequado: Má aplicação dos conceitos pode levar a sistemas rígidos.

Padrões de Projeto

Padrões de projeto são soluções reutilizáveis para problemas comuns no desenvolvimento de software orientado a objetos. Exemplos incluem:

  • Singleton: Garante que uma classe tenha apenas uma instância.
  • Factory Method: Define uma interface para criar objetos, mas permite que subclasses decidam qual classe instanciar.
  • Observer: Define uma dependência um-para-muitos entre objetos, de modo que quando um objeto muda de estado, todos os seus dependentes são notificados.

Princípios SOLID
