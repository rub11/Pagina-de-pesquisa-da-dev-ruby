🔑 Senhas Seguras e Hash

🔐 Por que senhas seguras importam?

Senhas são a primeira linha de defesa digital. Uma senha fraca pode permitir que invasores acessem dados confidenciais ou comprometam sistemas inteiros.

🛡️ Boas práticas para criar senhas seguras

  • 🔠 Misture letras maiúsculas e minúsculas
  • 🔢 Use números e símbolos
  • 📏 Tenha pelo menos 12 caracteres
  • 🚫 Evite palavras comuns ou nomes pessoais
  • 🔄 Use senhas únicas para cada serviço

💾 O que é Hash?

Hash é uma função matemática que transforma dados (como senhas) em uma sequência fixa de caracteres. Mesmo pequenas alterações geram resultados totalmente diferentes.

🔐 Por que usamos hash em senhas?

Senhas nunca devem ser armazenadas em texto puro. Com hash, mesmo que um banco de dados seja comprometido, os atacantes não verão a senha original.

🔄 Hashes comuns usados em segurança

  • 🧂 SHA-256 – comum, mas deve ser usado com salt
  • 💪 bcrypt – resistente a ataques de força bruta
  • ⛏️ Argon2 – atual padrão recomendado

🧂 O que é Salt?

Salt é um valor aleatório adicionado à senha antes da aplicação do hash. Isso impede que duas senhas iguais gerem hashes iguais e dificulta ataques por dicionário.

📌 Exemplo de uso com bcrypt

Ao registrar o usuário, a senha é “salteada” e depois convertida com bcrypt. Na autenticação, a senha digitada é comparada com o hash salvo.

🔍 Como verificar senhas fortes?

  • 🔐 Use geradores de senhas fortes
  • 🧠 Evite repetir senhas
  • 📱 Utilize autenticação de dois fatores (2FA)
  • 🔁 Troque senhas periodicamente em serviços críticos

📊 Curiosidade:

A senha "123456" ainda é uma das mais usadas no mundo 🌍. Nunca subestime o valor de uma senha bem construída!
