Context API no React

Compartilhando estado global de forma simples e elegante

O que é Context API?

A Context API é uma ferramenta nativa do React que permite compartilhar dados entre componentes sem a necessidade de prop drilling — ou seja, sem precisar passar props manualmente por todos os níveis da árvore de componentes.

Quando usar Context?

  • Quando múltiplos componentes precisam acessar o mesmo dado.
  • Para armazenar temas, autenticação, preferências do usuário, etc.
  • Quando você quer evitar o uso de bibliotecas externas para estados simples.

Como usar Context API

1. Criar o contexto:

{`import { createContext } from 'react'; export const TemaContext = createContext();`}

2. Criar o Provider:

{`import React, { useState } from 'react'; import { TemaContext } from './TemaContext'; export const TemaProvider = ({ children }) => { const [tema, setTema] = useState('dark'); return ( {children} ); };`}

3. Usar em um componente:

{`import { useContext } from 'react'; import { TemaContext } from './TemaContext'; function BotaoTema() { const { tema, setTema } = useContext(TemaContext); return ( ); }`}

Vantagens

  • É nativo do React (sem dependências externas).
  • Fácil de implementar e entender.
  • Ideal para estados globais simples.

Limitações

  • Re-renderizações em massa podem ocorrer se o contexto mudar com frequência.
  • Não é ideal para estados complexos (como listas grandes ou cache).

Conclusão

O Context API é uma poderosa funcionalidade do React que permite compartilhamento de estado global com facilidade. Ideal para muitos casos de uso, principalmente quando não há necessidade de uma solução mais robusta como Redux.
