Pull Requests e Revisão de Código

Colaboração, controle e qualidade no desenvolvimento em equipe

O que é um Pull Request?

Um Pull Request (PR) é uma solicitação para integrar mudanças de uma branch em outra (geralmente para main), muito usado em plataformas como GitHub, GitLab ou Bitbucket. Ele permite que outros revisem e aprovem antes da fusão final.

Como criar um Pull Request

Depois de subir seu código para uma branch remota:

git push origin minha-feature

Em seguida, vá até o repositório no GitHub e clique em "Compare & pull request".

Adicione título e descrição do que foi feito, e envie o PR para revisão.

Etapas de Revisão de Código

  1. Leitura do código: verificar clareza, lógica e boas práticas.
  2. Comentários: sugerir melhorias, apontar bugs, elogiar boas soluções.
  3. Testes: garantir que o código não quebre funcionalidades existentes.
  4. Aprovação: quando estiver tudo certo, aprovar e fazer o merge.

Fazendo o Merge

Após aprovação, o autor ou revisor pode fazer o merge. Existem três formas principais:

  • Merge Commit: mantém histórico detalhado.
  • Squash and Merge: compacta tudo em um único commit.
  • Rebase and Merge: reescreve a linha do tempo para manter linearidade.

Boas práticas em PRs

  • Crie PRs pequenos e objetivos
  • Descreva claramente o que foi feito
  • Responda e atualize de acordo com feedbacks
  • Utilize checklists e labels para organização
